1. Project Overview
ECLIPSE is a two-component malaria diagnostic system designed for point-of-care use in
resource-limited settings. The system detects Plasmodium falciparum lactate dehydrogenase (PfLDH), a
protein secreted into patient blood during active malaria infection, using an engineered protein biosensor
that converts target binding directly into a luminescence signal.
Component 1 — LOCKR Biosensor: A modified lucCage protein with a de novo designed
PfLDH-binding peptide grafted into its latch region. When PfLDH binds the binder, it pulls the latch open
→ exposes SmBiT → lucKey binds → NanoLuciferase reconstitutes → light output.
Component 2 — Coffee-Ring Signal Amplification: Evaporation of a droplet concentrates analyte and
signal at the ring perimeter, amplifying sensitivity without additional equipment.
Why PfLDH? PfLDH is secreted at ~100 ng/mL during active infection, species-specific isoforms
discriminate P. falciparum from other Plasmodium species, and it is absent from healthy human blood. It
is an established clinical target used in commercial RDTs.
Why LOCKR? The lucCage system (Quijano-Rubio et al. 2021, Baker Lab) provides a modular, tunable
biosensor scaffold. The cage protein normally holds SmBiT sequestered in its latch. Grafting a custom
binder into the latch converts the cage into a sensor for any protein of interest. Signal amplitude is
tunable by modifying latch stability and binder affinity.

4. Design Problems and Solutions
4.1 Critical: Charged Residues Destroyed K_CK
After threading the original RFDiffusion binder into latch position 327, thermodynamic analysis revealed a
fatal problem. The original binder LISDAELEAIFAEELDC has 6 D/E residues at positions 4, 6, 8, 13, 14,
16 — all sitting at the lucKey-cage interface. This causes electrostatic repulsion that collapses K_CK:
Calculation:
  6 charged residues x 0.8 kcal/mol penalty each = 4.8 kcal/mol
  Original dG_CK = -RT * ln(K_CK) = -0.592 * ln(1e8) = -10.9 kcal/mol
  Grafted  dG_CK = -10.9 + 4.8 = -6.1 kcal/mol
  K_CK_grafted  = exp(6.1 / 0.592) = ~3e-12
Effect on signal:
  Signal = PfLDH-binding x lucKey-binding
  Original: excellent PfLDH binding x K_CK=3e-12 (lucKey never binds) = ZERO
  Optimized: good PfLDH binding     x K_CK=1e-8  (lucKey binds)       = SIGNAL
Solution: Mutate all non-interface D/E to Alanine. Interface residues (positions 1, 2, 11, 12, 15 — PfLDH
contacts from RFDiffusion design) preserved.
Mutations: D4A, E6A, E8A, E13A, E14A, D16A
Original:  LISDAELEAIFAEELDC
Optimized: LISAAALAAIFAAALAC

4.3 AlphaFold3 Complex Low ipTM — Correct Interpretation
Submitting lucCage + PfLDH to AlphaFold3 (alphafoldserver.com, which runs AF3) returned ipTM = 0.21
(v1.0) and 0.15 (v2.2). This initially appeared concerning but has a clear mechanistic explanation.
AF3 predicts the stable complex structure. But LOCKR works through a conformational change: PfLDH
grabs the binder in the closed-latch state, then the latch opens. AF3 sees a closed cage where the binder
is partially occluded — the wrong state to evaluate binding. ADCP peptide docking (which isolated the
binder as a flexible peptide) is the appropriate tool and gave strong results.

After completing v1.0, we asked whether binding could be improved without full redesign. The avidity
principle: two copies of the same binder on one molecule create cooperative binding — once one copy
captures PfLDH, the second is pre-positioned to engage. Both binders simultaneously apply allosteric
force to open the latch.
Latch (325-359):  LISAAALAAIFAAALAC + G + LISAAALAAIFAAALAC
                  |____ Binder 1 ____|   |____ Binder 2 ____|
                  17 aa               1aa  17 aa
                  Total = 35 aa = exactly 35-residue latch
Result: ΔΔG = -4.6 kcal/mol, Kd improved 2369x (100 pM to 42 fM).
4.5 v2.2 Raw PDB Had Severe Clashes
After manual sequence insertion of the tandem binder, PyRosetta scored the raw PDB at approximately
+900 REU — atoms overlapping. FastRelax (5 rounds, Cartesian) resolved all clashes: +900 to -1470.75
REU (improvement of 2371 REU).

Script 1 — PyRosetta Threading: Original Binder
Scans positions 325-343, threads LISDAELEAIFAEELDC at each, scores with REF2015.
from pyrosetta import *
from pyrosetta.toolbox import mutate_residue
init(options="-mute all")
BINDER_SEQ = "LISDAELEAIFAEELDC"
LATCH_START = 325
LATCH_END   = 359
SMBITT_POS  = 312
SMBITT_SEQ  = "VTGYRLFEEIL"
template_pose = pose_from_pdb("lucCage_template.pdb")
scorefxn = get_fa_scorefxn()
results = []
for start_pos in range(LATCH_START, LATCH_END):
    if start_pos + len(BINDER_SEQ) - 1 > LATCH_END:
        continue
    pose = Pose()
    pose.assign(template_pose)
    for i, aa in enumerate(BINDER_SEQ):
        rpos = start_pos + i
        if SMBITT_POS <= rpos < SMBITT_POS + len(SMBITT_SEQ):
            continue
        mutate_residue(pose, rpos, aa)
    score = scorefxn(pose)
    results.append({'pos': start_pos, 'score': score})
results.sort(key=lambda x: x['score'])
for i, d in enumerate(results[:5]):
    print(f"{i+1}. Position {d['pos']} | Score: {d['score']:.2f}")

Script 8 — LOCKR Thermodynamic Fold-Change Model
Models the three-state LOCKR equilibrium to predict fold-change vs PfLDH concentration.
import numpy as np
K_open = 1e-3    # cage opening equilibrium
K_CK   = 1e-8    # lucKey-cage Kd
lucKey = 500e-9  # assay lucKey concentration
# Fold-change equation:
# f_base   = K_open         / (1 + K_open         + [lucKey]/K_CK)
# f_signal = K_open_eff     / (1 + K_open_eff     + [lucKey]/K_CK)
# K_open_eff = K_open * (1 + pull * theta)
# theta = [PfLDH] / ([PfLDH] + Kd)
# FC = f_signal / f_base
def fold_change(pfldh, Kd, pull):
    theta    = pfldh / (pfldh + Kd)
    Koe      = K_open * (1 + pull * theta)
    f_base   = K_open / (1 + K_open + lucKey/K_CK)
    f_signal = Koe    / (1 + Koe    + lucKey/K_CK)
    return f_signal / f_base
Output (selected):
v1.0 (Kd=100pM, pull=10):  max FC=11x   EC50=0.10 nM  LOD~10 pM
v1.0 (Kd=100pM, pull=20):  max FC=21x   EC50=0.10 nM  LOD~10 pM
v2.2 (Kd=42fM,  pull=10):  max FC=11x   EC50=~0 nM    LOD~0 pM
v2.2 (Kd=42fM,  pull=20):  max FC=21x   EC50=~0 nM    LOD~0 pM
v2.2 (Kd=42fM,  pull=30):  max FC=31x   EC50=~0 nM    LOD~0 pM
Fold-change ceiling is set by K_open/K_CK (cage mechanics), not binder affinity. v2.2 advantage is
sensitivity: sensor is always saturated at clinical PfLDH concentrations because Kd (42 fM) is far below
clinical levels (~1 nM+).

8. v1.1 Latch Mutation Study — Attempted Improvement
After completing the v1.0 and v2.2 pipelines, we attempted to apply the Baker Lab strategy of
latch-destabilizing mutations to v1.0, creating a v1.1 variant. The goal was to test whether lowering the
latch opening energy cost (increasing K_open) would improve fold-change independently of the avidity
improvement in v2.2. This section documents the complete attempt, including why it did not produce a
meaningful improvement and why we decided not to synthesize v1.1.
8.1 Motivation and Candidate Positions
Quijano-Rubio et al. describe mutations V223S and I238S that destabilize the latch closed state. In our
sequence, position 223 = R and 238 = E — the numbering does not transfer because our lucCage variant
has different GSGSGS linker positions. We searched our own latch (325-359) for safe I/V candidates:
Latch (325-359): ERLISAAALAAIFAAALACRLIREAAAASEKISRE
                                            ^^        ^^
                                          I346       I356   <-- candidates
Positions 346 and 356 are isoleucine residues downstream of the binder, in the native latch region.
Mutating to serine introduces a hydroxyl that is incompatible with the hydrophobic latch core, specifically
destabilizing the closed conformation without affecting the binder or SmBiT.

8.5 Why Latch Mutations Don't Improve Performance Here
The results show fold-change and EC50 are essentially identical across all three v1.1 scenarios and v1.0.
In the optimistic case, fold-change actually decreases slightly. The reason is in the denominator of the
fold-change equation:
f_base = K_open / (1 + K_open + [lucKey]/K_CK)
       = 1e-3   / (1 + 0.001  + 500nM/10nM)
       = 1e-3   / (1 + 0.001  + 50)
       = 1e-3   / 51.001
The [lucKey]/K_CK term = 50, which completely dominates the denominator.
K_open at 1e-3 is negligible compared to 50.
Even at K_open=3e-2 (30x increase): 1 + 0.03 + 50 = 51.03 -- barely changed.
Conclusion: the fold-change ceiling is set by [lucKey]/K_CK, not K_open.
Latch mutations only help if [lucKey]/K_CK is reduced to a comparable magnitude.
For latch mutations to produce a meaningful improvement, the assay would need to run at [lucKey] =
10-50 nM instead of 500 nM, bringing [lucKey]/K_CK to 1-5 rather than 50. This can be tested in wet lab
by titrating lucKey concentration — no additional gene synthesis required.

11.3 Luminescence Assay
Reagents:
  100 nM lucCage (v1.0 or v2.2)
  500 nM lucKey
  10 uM coelenteramide (substrate)
  Buffer: 25 mM HEPES pH 7.4, 150 mM NaCl, 0.01% Tween-20
Protocol:
  1. Mix lucCage + lucKey + coelenteramide
  2. Measure L_baseline
  3. Add PfLDH: 0, 1pM, 10pM, 100pM, 1nM, 10nM, 100nM, 1uM
  4. Incubate 10-30 min at 37C per point
  5. Measure L_signal
  6. Fit: FC = 1 + (FC_max - 1) * [PfLDH] / ([PfLDH] + EC50)
Success criteria:
  > 50x  -- v2.2 avidity principle fully demonstrated
  > 10x  -- design validated
  5-10x  -- try lucKey titration (50 nM instead of 500 nM)
  < 5x   -- structural re-evaluation needed
11.4 Measuring Pull Strength
Pull strength is the allosteric coupling constant — how efficiently binder engagement translates to latch
opening. It cannot be measured directly but is back-calculated from the luminescence titration curve.
from scipy.optimize import curve_fit
import numpy as np
K_open = 1e-3
K_CK   = 1e-8
lucKey = 500e-9
Kd     = 100e-12  # v1.0
def fold_change_model(pfldh, pull_strength):
    theta    = pfldh / (pfldh + Kd)
    K_oe     = K_open * (1 + pull_strength * theta)
    f_base   = K_open / (1 + K_open + lucKey/K_CK)
    f_signal = K_oe   / (1 + K_oe   + lucKey/K_CK)
    return f_signal / f_base
# pfldh_conc and fc_measured = your experimental data
popt, _ = curve_fit(fold_change_model, pfldh_conc, fc_measured, p0=[10])
pull_strength = popt[0]
print(f"Pull strength = {pull_strength:.1f}")
